# Naming Convention Rules
- Always prefix tables with their medallion (e.g., `bronze_`, `silver_`, `gold_`) level.
- Table metadata must be fully populated, including descriptions for the table and each column.
- If creating catalogs or schemas in this demo, prefix the name with a unique identifier (e.g., your initials) to avoid conflicts with other participants.

# Writing Style
- Do not use em-dashes (—), en-dashes (–), or semicolons (;) in prose, code comments, table descriptions, or commit messages. Use hyphens or split into separate sentences instead.

# Process
- Ask for confirmation if objects or destinations are unclear.
